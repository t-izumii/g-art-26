# アート実装解説 — NAGARE（流れ）

p5.js で描く生成アニメーション「**流れ（NAGARE）**」の実装解説。
オーブ（粒）とリボン（葉・流線）の群れが、画面を斜めに流れ続ける有機的なモーションを作る。

> LEDキューブの面定義・モード切替・キャンバス生成などはテンプレート側の仕組みなので省略。
> ここでは `GraphicsLayer` クラスを中心としたアート描画ロジックだけを扱う。

各レイヤーは `createGraphics(size, size)` のオフスクリーン正方形バッファを持ち、その上に流れアニメーションを描く。

---

## 1. デザインパラメータ

アニメーションの見た目は `NAGARE_DEFAULTS` で一括管理される。

```js
NAGARE_DEFAULTS = {
  axis,        // 流れの基準角度（進行方向は -90° + axis）
  loopSec,     // ループ周期（秒）。これでアニメが完全ループする
  spread,      // 横方向の広がり幅
  scale,       // 全体スケール
  variation,   // 個体サイズのばらつき量
  introSec,    // 登場アニメの時間
  density,     // オブジェクト密度（生成数に影響）
  palette,     // カラーパレット（hex配列）
  weights,     // 各色の出現重み
}
```

- **配色**: 中身は `palette`（カラフルな粒）、背景は `NAGARE_CREAM`（クリーム色）、輪郭は `NAGARE_OUTLINE`（濃い茶）。絵本のような手描きトーンを作る要素。
- 色は `weightedIndex()` が `weights` に基づく重み付き抽選で選ぶ（出やすい色／出にくい色を制御）。

---

## 2. シーン構築 — buildScene()

レイヤーごとに `randomSeed` / `noiseSeed` を `index` でずらし、**再現可能だが各レイヤーで異なる**配置を生成する。

生成数はサイズと密度から決定：
```js
shapes = max(60, round(density * size * size / (760 * 1180)))
```

各オブジェクトは2タイプ：

| タイプ | 割合 | 形 | 特徴 |
|--------|------|----|------|
| **オーブ (orb)** | 約40% | 楕円の粒 | `aspect` で縦横比をわずかに変える |
| **リボン (ribbon)** | 残り | 帯・流線 | 約18%が**葉 (leaf)** になり専用色に。`profile` で太さ分布が変化 |

各オブジェクトが持つ主なプロパティ：

```js
{
  p0,        // 初期位相（流れの開始位置をずらす）
  laneN,     // 横方向レーン位置 (-1..1)
  swayPh,    // 揺れの位相
  swayAmp,   // 揺れの振幅
  orbPh,     // パルス/カール用の位相
  sz,        // サイズ揺らぎ (-1..1)
  colIdx,    // 色インデックス
  // リボン専用: angOff, lenC, widC, curlF, profile
}
```

`profile`（0/1/2）はリボン先端の太さの減衰カーブを切り替え、葉・しずく・流線など見た目を多様化させる。

---

## 3. フレーム描画 — draw()

毎フレームの流れの作り方。

### 3.1 ループ位相

```js
loopPhase = (frameCount % loopFrames) / loopFrames;  // 0→1 を周期的に繰り返す
theta     = TWO_PI * loopPhase;                        // 周回角
```

`loopPhase` を全オブジェクトで共有することで、`loopSec` 秒で**完全にループ**するアニメーションになる（NEORT 等のループ再生に適する）。

### 3.2 個体の位置計算

進行方向 `dir` と直交方向 `perp` を基準に、各オブジェクトの位置を求める：

```js
pp    = (s.p0 + loopPhase) % 1;          // 個体ごとの進行率
along = (pp - 0.5) * span;               // 流れ方向の移動量 (span = H * 1.6)
lane  = s.laneN * bandHalf               // 横位置
        + sin(theta + s.swayPh) * s.swayAmp;  // ＋ゆらゆら揺れ
px = cx + dir.x*along + perp.x*lane;
py = cy + dir.y*along + perp.y*lane;
```

`span` を画面より大きく取ることで、オブジェクトは**画面外まで流れて折り返す**（見切れながら連続的に流れる演出）。

### 3.3 登場とパルス

- **登場アニメ**: `introScale`（`nagSmoothstep` でイージング）で、開始時にふわっと出現。
- **パルス**: `1 + 0.05 * sin(frameCount*0.03 + 位相)` で個体ごとに微妙な拡縮を加え、生き物のような揺らぎを出す。

### 3.4 描画

- **オーブ**: `ellipse` を `aspect` 付きで描く
- **リボン**: `ribbonOutline()` で輪郭点列を生成し `beginShape`〜`endShape(CLOSE)` で塗り。葉の場合は中央に葉脈ラインを追加描画。
- すべて `NAGARE_OUTLINE` 色の輪郭線付きで、線幅はサイズ比例。

## 4. 1つ1つの「形」はどうやってできている？

オブジェクトは2種類。**オーブは超かんたん**、**リボンが作品の肝**。

### 4.1 オーブ（粒）= 楕円1個

p5 の `ellipse` を1個描くだけ。

```js
g.ellipse(px, py, rEff * 2 * s.aspect, rEff * 2);
//                     ↑ aspect = 0.82〜1.18
```

`aspect` で縦横比をちょっとだけ変えているので、真ん丸ではなく「少しゆがんだ卵型」になる。これだけ。

---

### 4.2 リボン（葉・流線）= 点をつないで自分で形を作る

リボンは「葉っぱ」や「筆の一画」のような形。p5 に既成の図形は無いので、
**点を1つずつ計算して、その点を線でつないで多角形にしている**（関数 `ribbonOutline()`）。

イメージは「**ほねを引く → 肉をつける → 周りを囲んで塗る**」の3ステップ。

#### ステップ1：背骨（中心線）を引く

進行方向に向かって、点を 22個 ならべて1本の線にする。
ただし真っすぐではなく、**真ん中をふくらませて弓なりに曲げる**（`curl`）＋ **時間で揺らす**（`wob`）。

```
真っすぐな線         curlで弓なりに         wobで揺らぐ
 ·····>          →    ·‥˙˙‥·    →     ·‿˙‾‿·
（根元）（先端）        （カーブ）           （うねうね）
```

#### ステップ2：各点に「太さ」をつける

背骨の点ごとに幅を決める。この**太さの分布カーブ＝`profile`** で、リボンの見た目が3種類に変わる。

```
profile=0（しずく型）   profile=1（葉型）      profile=2（先太り）
 ◣＿＿               ◢▆◣               ＿＿◢
根元が太く先細り        中央がふくらむ         先っぽが太い
```

#### ステップ3：背骨の左右にふくらませて、ぐるっと囲む

各点で「背骨に対して真横の向き」を求め、太さの半分だけ**左右にずらした点**を作る。
その左右の点を、根元→先端→根元と1周つなぐと、閉じた輪郭になる。
**先っぽと根元は、角ばらないよう円のカーブでなめらかに丸める**。

```
   背骨だけ            左右にふくらます        ぐるっと囲んで閉じる
                       左●  ●  ●
    ·                    \ | /             ╭───────╮
   ·中心線·     →       ●─背骨─●     →    （  塗りつぶし  ）
    ·                    / | \             ╰───────╯
                       右●  ●  ●          ← 両端は丸い
```

これを p5 で塗れば、リボン1個のできあがり：

```js
g.beginShape();
for (k of geom.out) g.vertex(k.x, k.y);  // 囲んだ輪郭点をぜんぶつなぐ
g.endShape(CLOSE);                        // 閉じて塗る
```

> **葉（leaf）のときだけ**、おまけで背骨の線（葉脈）も上から描き足す。

#### まとめ図

```
オーブ  →  ◯ 楕円1個（aspectでゆがみ）

リボン  →  ①背骨を引く（22点・弓なり・揺れ）
          ②太さをつける（profileで3種類）
          ③左右にふくらませ、丸く囲んで塗る
```

---

### 補助に使っている計算関数

- `nagSmoothstep(t)` — なめらかな立ち上がり（登場アニメ用）
- `nagEaseOutBack(t)` — いったん行き過ぎて戻るバウンド
- `nagAngTo(from,to)` — 角度の差を計算（端を円で丸めるときに使用）

## 5. ライブ調整フック

HTML側のUIパネルから生成パラメータをリアルタイム変更するための関数群：

| 関数 | 役割 |
|------|------|
| `nagareUpdateParam(name, value)` | 数値パラメータを更新 |
| `nagareUpdateColor(idx, value)` | パレット色を更新 |
| `nagareRegenerate()` | 全レイヤーを再構築（`buildScene` 再実行）|
| `nagareReplayIntro()` | 登場アニメを再生し直す |
| `nagareTogglePanel()` | パネルの開閉 |

---

## 6. 表現のまとめ

- **完全ループ**: `loopPhase` 共有で `loopSec` 秒のシームレスループ
- **多様性**: シード分離＋重み付き配色＋オーブ/リボン/葉の混在で、同じレイヤーでも豊かなバリエーション
- **有機性**: カール・うねり・揺れ・パルスの重ね合わせで生き物のような流れ
- **絵本トーン**: クリーム背景＋濃い輪郭線＋彩度高めパレット
